1.3.0 Design Implementation & Constraints
--------------------------------------------------------------------
<Insert Application Name> v1.0 Design Implementation & constraints

-R1.1: The application must be delivered to the client no later than the 4th December 2014

-R1.2: The application must be compatible with Android devices running Android 4.1.1 and above

-R1.3: The application must be compatible with a variety of screen sizes, and scale all elements accordingly.

-R1.4: The application colour scheme must match that of the Cadence health website

-R1.5: The application must follow the Android Design guidelines and conform to common UI patterns found at 
http://developer.android.com/distribute/essentials/quality/core.html. This includes
	-The application does not redefine the expected function of a system icon (such as the "home" or "back " buttons)
	-The application does not misuse Android UI patterns, that would make a standard user become confused. 

-R1.6: The application must function normally when installed on an SD card

-R1.7: The application must request the absolute minimum permissions that it needs to operate normally. 

-R1.8: The application should support both landscape and portrait orientations

-R1.9: The application must adhere to the terms of the Google play developer content policy found at https://play.google.com/about/developer-content-policy.html
